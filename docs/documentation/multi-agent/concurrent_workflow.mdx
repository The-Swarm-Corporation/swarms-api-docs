**Swarm Type**: `ConcurrentWorkflow`

## Overview

The ConcurrentWorkflow swarm type runs independent tasks in parallel, significantly reducing processing time for complex operations. This architecture is ideal for tasks that can be processed simultaneously without dependencies, allowing multiple agents to work on different aspects of a problem at the same time.

Key features:
- **Parallel Execution**: Multiple agents work simultaneously
- **Reduced Processing Time**: Faster completion through parallelization
- **Independent Tasks**: Agents work on separate, non-dependent subtasks
- **Scalable Performance**: Performance scales with the number of agents

## Use Cases

- Independent data analysis tasks
- Parallel content generation
- Multi-source research projects
- Distributed problem solving

## Tutorial: Build a Competitive Intelligence Dashboard

In this tutorial, you'll build a competitive intelligence system that analyzes multiple competitors simultaneously. Instead of researching one company at a time, ConcurrentWorkflow lets all analyses run in parallel - turning what could be a lengthy sequential process into a fast, efficient operation.

### What You'll Build

A parallel competitor analysis system that:
1. Researches **5 competitors simultaneously**
2. Analyzes each across multiple dimensions (products, pricing, market position)
3. Aggregates results into a unified dashboard view
4. Compares execution time vs. sequential approach

### Prerequisites

- Python 3.8+
- A Swarms API key ([get one here](https://swarms.world))
- `requests` library (`pip install requests`)

### Step 2: Define Your Competitor Analysis Agents

In ConcurrentWorkflow, each agent works **independently and in parallel**. Design agents so they don't depend on each other's output.

```python
def create_competitor_agents(competitors: list[str]) -> list[dict]:
    """
    Create one analysis agent per competitor.
    Each agent analyzes a single competitor independently.

    Args:
        competitors: List of competitor company names

    Returns:
        List of agent configurations
    """

    agents = []

    for competitor in competitors:
        agent = {
            "agent_name": f"{competitor} Analyst",
            "description": f"Analyzes {competitor}'s market position and strategy",
            "system_prompt": f"""You are a competitive intelligence analyst specializing in {competitor}.

Analyze {competitor} across these dimensions:

1. **COMPANY OVERVIEW**
   - Founded, headquarters, employee count
   - Recent funding/valuation (if applicable)
   - Key leadership

2. **PRODUCT ANALYSIS**
   - Core products/services
   - Recent launches or updates
   - Unique features vs. competitors
   - Known limitations or weaknesses

3. **MARKET POSITION**
   - Target customer segments
   - Estimated market share
   - Geographic presence
   - Key partnerships

4. **PRICING STRATEGY**
   - Pricing model (subscription, usage-based, etc.)
   - Price points for main offerings
   - Free tier or trial availability
   - Enterprise pricing approach

5. **STRENGTHS & WEAKNESSES**
   - Top 3 competitive strengths
   - Top 3 vulnerabilities to exploit

6. **RECENT NEWS**
   - Notable announcements (last 6 months)
   - Strategic moves or pivots

Format your response with clear headers and bullet points.
Be specific with numbers and facts where possible.""",
            "model_name": "gpt-4o",
            "max_loops": 1,
            "temperature": 0.3  # Lower temperature for factual research
        }
        agents.append(agent)

    return agents
```

### Step 3: Build the Concurrent Analysis Function

```python
def analyze_competitors(
    competitors: list[str],
    industry: str,
    your_company: str
) -> dict:
    """
    Analyze multiple competitors in parallel.

    Args:
        competitors: List of competitor company names
        industry: The industry context
        your_company: Your company name for context

    Returns:
        dict: API response with parallel analysis results
    """

    agents = create_competitor_agents(competitors)

    swarm_config = {
        "name": f"Competitive Intelligence: {industry}",
        "description": f"Parallel competitor analysis for {your_company}",
        "swarm_type": "ConcurrentWorkflow",
        "task": f"""Conduct competitive intelligence analysis for {your_company}
in the {industry} industry.

Each analyst should focus on their assigned competitor and provide
a comprehensive analysis. We are analyzing: {', '.join(competitors)}.

Provide actionable insights that {your_company} can use to compete effectively.""",
        "agents": agents,
        "max_loops": 1
    }

    response = requests.post(
        f"{API_BASE_URL}/v1/swarm/completions",
        headers=headers,
        json=swarm_config,
        timeout=180  # Parallel execution is faster but allow buffer
    )

    if response.status_code == 200:
        return response.json()
    else:
        raise Exception(f"API Error: {response.status_code} - {response.text}")
```

        results.append({
            "industry": analysis['industry'],
            "result": result
        })

    return results

### Key Takeaways

1. **Parallel = Speed**: 5 competitors analyzed in the same time as 1. The more agents, the bigger the time savings.

2. **Independent Tasks Only**: ConcurrentWorkflow agents cannot see each other's output. Design tasks that don't require inter-agent communication.

3. **Same Task, Different Focus**: All agents receive the same task but their system prompts direct them to different targets.

4. **Consistent Output Format**: Use identical output structure across agents to make aggregation easier.

5. **Cost Efficiency**: Parallel execution doesn't increase cost - you pay the same tokens whether sequential or concurrent.

### When to Use ConcurrentWorkflow vs. SequentialWorkflow

| Use ConcurrentWorkflow When | Use SequentialWorkflow When |
|----------------------------|----------------------------|
| Tasks are independent | Tasks have dependencies |
| Speed is critical | Order matters |
| Analyzing multiple items | Building on previous output |
| No inter-agent communication needed | Agents need to see prior work |
| Aggregating diverse perspectives | Creating a pipeline/chain |

## Best Practices

- Design independent tasks that don't require sequential dependencies
- Use for tasks that can be parallelized effectively
- Ensure agents have distinct, non-overlapping responsibilities
- Ideal for time-sensitive analysis requiring multiple perspectives
