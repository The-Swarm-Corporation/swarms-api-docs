## What This Example Shows

- How to drive `POST /v1/advanced-research/completions` from a structured prompt that yields a 1-page investor memo

## Why This Matters

Equity analysts at hedge funds and asset managers spend the bulk of Monday writing the same artifact: a one-page weekly memo on each name in their book — earnings preview, catalyst calendar, sell-side delta, news that moved the tape, and a clear recommendation. Producing one memo by hand takes a junior analyst 2-4 hours and the output is only as fresh as their last manual sweep through filings, news, and broker notes. The Advanced Research endpoint does that sweep for you: a director agent plans the sub-questions, worker agents run external searches and read primary sources, and the final synthesis lands as a cited memo your PM can actually trust. You ship a watchlist's worth of memos before the open instead of by Tuesday lunch.

import json

import requests
from dotenv import load_dotenv

load_dotenv()

API_KEY = os.getenv("SWARMS_API_KEY")

## Step 2: Define the Memo Template as the Task

The Advanced Research director agent listens to the task. Give it the exact section headers you want in your memo and it will plan the sub-research accordingly. This is where you encode your house style.

```python
TICKER = "NVDA"
QUARTER = "FQ3"

memo_task = f"""
Produce a one-page institutional-grade investor memo on {TICKER} ahead of its
{QUARTER} earnings release. Audience: a portfolio manager who already owns the
name and needs to decide whether to add, hold, or trim before the print.

Use the following exact section structure. Every factual claim must include an
inline citation in the form [source: URL].

# {TICKER} — {QUARTER} Earnings Preview

## 1. Buy-Side Consensus vs. Sell-Side Consensus
Top-line revenue, EPS, and data center segment expectations. Highlight any
whisper-number gap.

## 2. Catalysts Into the Print
Three to five catalysts ranked by expected impact. Include guidance posture,
product-cycle inflection points, and any pending regulatory items.

## 3. Risks
Two to three risks that could compress the multiple or guide-down. Be specific
(geographic exposure, customer concentration, inventory, FX).

## 4. Sell-Side Delta (last 2 weeks)
Notable target price changes, upgrades, downgrades, and initiations. Name the
firms.

## 5. Recommendation
ADD / HOLD / TRIM with a one-sentence rationale tied to the items above. Do
not hedge.

Cite every claim. If a claim cannot be sourced, mark it [unsourced] rather
than fabricating.
""".strip()
```

<Note>
The phrase "If a claim cannot be sourced, mark it [unsourced]" is load-bearing. Without it, the synthesis agent will paper over gaps in retrieval with plausible-sounding statements. Forcing an explicit `[unsourced]` token makes hallucinations visible to your reviewer.
</Note>

## Step 3: Configure the Research Run

The `config` block controls model choice and search depth. For investor memos you want the most capable models on both director and worker tiers — accuracy beats throughput.

```python
payload = {
    "config": {
        "name": f"{TICKER} Weekly Investor Memo",
        "description": (
            f"Pre-earnings investor memo on {TICKER}. Director plans the "
            f"research; workers run multi-query Exa searches across filings, "
            f"news, and analyst notes; synthesis produces a cited 1-pager."
        ),
        "director_model_name": "anthropic/claude-opus-4-8",
        "worker_model_name": "anthropic/claude-opus-4-8",
        "director_max_tokens": 8000,
        "max_loops": 2,
        "exa_search_num_results": 6,
        "exa_search_max_characters": 1500,
    },
    "task": memo_task,
}
```

Key knobs:

| Field | Effect |
| --- | --- |
| `max_loops: 2` | Director re-plans once after seeing first-pass results — catches missing catalysts |
| `exa_search_num_results: 6` | Six sources per sub-query. Good middle ground; bump to 8-10 for compliance work |
| `exa_search_max_characters: 1500` | Lets the worker actually read the article instead of guessing from the snippet |
| `director_max_tokens: 8000` | Headroom for the director to write a detailed synthesis with citations |

## Step 4: Run It

```python
response = requests.post(
    f"{BASE_URL}/v1/advanced-research/completions",
    headers=headers,
    json=payload,
    timeout=600,
)
response.raise_for_status()
result = response.json()

print(json.dumps(result, indent=2)[:2000])
```

## Step 5: Extract the Memo and Sources

The response carries the synthesis under `outputs` and the URLs the workers actually fetched under `sources`. Pull them apart so you can publish the memo to your CMS or Slack channel and audit citations separately.

```python
memo = result.get("outputs")
sources = result.get("sources", [])

if isinstance(memo, list):
    memo = "\n\n".join(str(item) for item in memo)

print("=" * 70)
print("MEMO")
print("=" * 70)
print(memo)

print("\n" + "=" * 70)
print(f"SOURCES ({len(sources)})")
print("=" * 70)
for i, src in enumerate(sources, start=1):
    url = src.get("url") if isinstance(src, dict) else src
    title = src.get("title", "") if isinstance(src, dict) else ""
    print(f"{i:>2}. {title}\n    {url}")

usage = result.get("usage", {})
billing = usage.get("billing_info", {})
print(f"\nTotal cost: ${billing.get('total_cost', 0):.4f}")
print(f"Execution time: {result.get('execution_time', 0):.1f}s")
```

## Step 6: Scale Across a Watchlist with Batch

Once one memo works, the batch endpoint runs your whole watchlist in parallel. Swap the loop for a single POST.

```python
WATCHLIST = ["NVDA", "AMD", "AVGO", "MSFT", "META"]

batch_payload = {
    "input_schemas": [
        {
            "config": {
                "name": f"{ticker} Weekly Investor Memo",
                "description": f"Pre-earnings investor memo on {ticker}.",
                "director_model_name": "anthropic/claude-opus-4-8",
                "worker_model_name": "anthropic/claude-opus-4-8",
                "max_loops": 2,
                "exa_search_num_results": 6,
                "exa_search_max_characters": 1500,
            },
            "task": memo_task.replace("NVDA", ticker),
        }
        for ticker in WATCHLIST
    ]
}

batch_response = requests.post(
    f"{BASE_URL}/v1/advanced-research/batch/completions",
    headers=headers,
    json=batch_payload,
    timeout=900,
)
batch_response.raise_for_status()

for ticker, memo_result in zip(WATCHLIST, batch_response.json()):
    print(f"\n### {ticker} memo (first 500 chars)\n")
    out = memo_result.get("outputs", "")
    if isinstance(out, list):
        out = "\n".join(str(x) for x in out)
    print(str(out)[:500])
```

<Note>
Batch runs concurrently server-side. A 5-name watchlist takes roughly the same wall clock as a single memo — not 5x longer.
</Note>
